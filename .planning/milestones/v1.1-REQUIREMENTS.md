# Requirements: HELIX

**Milestone:** v1.1 Growth, Polish & Quality
**Defined:** 2026-03-02
**Core Value:** Users can stake tokens for a chosen duration, earn T-shares proportional to their commitment, and receive daily inflation rewards — the complete stake-lock-earn lifecycle must work trustlessly on-chain.

## v1.1 Requirements

Requirements for the Growth, Polish & Quality milestone. Each maps to roadmap phases.

### Frontend Polish

- [x] **POLISH-01**: Public IDL (`app/web/public/idl/helix_staking.json`) is synchronized with Phase 8.1 ClaimConfig fields (`bpdFinalizeStartTimestamp`, `bpdOriginalUnclaimed`, `BpdBatchFinalized` event)
- [x] **POLISH-02**: Reward preview in stake-card and penalty-calculator applies loyalty multiplier (`applyLoyaltyMultiplier`) and shows accurate payout for mature stakes
- [x] **POLISH-03**: BPD status component shows "Awaiting 24h Observation Window" state between `bpdCalculationComplete=true` and `seal_bpd_finalize` execution
- [x] **POLISH-04**: Dashboard shows a protocol paused banner when `is_paused` (`GlobalState.reserved[1] == 1`) is set, blocking stake/unstake/claim interactions

### Referral System

- [x] **REF-01**: User can generate a referral link from the dashboard containing their wallet address as the referrer
- [x] **REF-02**: User who stakes via a referral link receives a +10% T-share bonus applied at stake creation (on-chain, via `create_stake`)
- [x] **REF-03**: Referrer receives a +5% T-share credit minted at the time the referred stake is created (both bonuses applied in same transaction)
- [x] **REF-04**: Self-referral is rejected on-chain with a custom error (`SelfReferral`) when `referrer == signer`
- [x] **REF-05**: Each referrer-referee pair can receive the referral bonus exactly once (duplicate prevention via `ReferralRecord` PDA)
- [x] **REF-06**: User can view their referral stats in the dashboard: total referrals made and total bonus T-shares earned via referrals

### NFT Badges

- [x] **BADGE-01**: Indexer tracks badge eligibility for all badge types per wallet address (stored in `badge_eligibility` table)
- [x] **BADGE-02**: User can view all earned and eligible badges in a badge gallery in the dashboard
- [x] **BADGE-03**: User can claim a soulbound Bubblegum V2 cNFT "First Stake" badge after creating their first stake
- [x] **BADGE-04**: User can claim a soulbound "365-Day Staker" badge after completing a stake with a committed duration of 365+ days
- [x] **BADGE-05**: User can claim a soulbound "Big Pay Day" badge after receiving a BPD distribution
- [x] **BADGE-06**: User can claim a soulbound stake-size tier badge (Shrimp ≥1K, Fish ≥10K, Dolphin ≥100K, Shark ≥1M, Whale ≥10M HELIX in a single stake) — one badge per tier threshold reached
- [x] **BADGE-07**: All badges are non-transferable (Bubblegum V2 `PermanentFreezeDelegate` with `authority: None`)

### Push Notifications

- [x] **NOTIF-01**: User can opt in to browser push notifications from the dashboard via Web Push API + service worker
- [x] **NOTIF-02**: User receives a push notification 7 days before a stake reaches its maturity date
- [x] **NOTIF-03**: User receives a push notification when a stake enters the 14-day late penalty grace window
- [x] **NOTIF-04**: User receives a push notification when they have claimable staking rewards available (accrued since last claim)
- [x] **NOTIF-05**: User receives a push notification on BPD phase transitions (finalization started, seal delay complete, distribution complete)
- [x] **NOTIF-06**: User can toggle individual notification event types on/off from a preferences panel in the dashboard
- [x] **NOTIF-07**: Dashboard is installable as a Progressive Web App (PWA manifest + install prompt) enabling push notifications on iOS via Home Screen

### Test Coverage

- [x] **TEST-01**: anchor-litesvm replaces deprecated bankrun in the Anchor test suite — all existing 165 tests pass with the new runner
- [x] **TEST-02**: Anchor program instruction test coverage ≥80% overall, with `math.rs` ≥95% and each instruction module ≥85%
- [x] **TEST-03**: Indexer service test coverage ≥80% — Vitest unit tests for event processors, polling worker, and all API route handlers
- [x] **TEST-04**: Frontend test coverage ≥80% — Vitest + React Testing Library tests for all components, custom hooks, and math utilities
- [x] **TEST-05**: Playwright E2E test suite covers stake creation, early unstake, on-time unstake, claim rewards, and free claim flows end-to-end
- [x] **TEST-06**: Playwright E2E test suite covers the full BPD 3-phase flow (finalize → seal → distribute) and referral bonus flow

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Mainnet deployment | Deferred — devnet only for v1.1 |
| On-chain referral leaderboard | Indexer-only tracking is sufficient; full leaderboard is v2 |
| Referral tiers/multipliers | Single bonus rate (10%/5%) for v1.1; tiered system adds complexity |
| Badge trading / marketplace | Badges are soulbound by design — non-transferable |
| Email notifications | Push-only for v1.1; email requires an external sending service and email collection |
| Mobile native app | Responsive web + PWA sufficient |
| Cross-chain bridging | Solana-native only |
