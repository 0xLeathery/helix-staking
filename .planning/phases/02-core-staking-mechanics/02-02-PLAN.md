**Account struct `CreateStake`:**
- `user`: Signer, mut (payer for PDA rent)

- `stake_account`: Account<StakeAccount>, init, payer=user, space=StakeAccount::LEN, seeds=[STAKE_SEED, user.key().as_ref(), &global_state.total_stakes_created.to_le_bytes()], bump

**Handler `create_stake(ctx, amount: u64, days: u16)`:**
1. Validate: `amount >= global_state.min_stake_amount` (error: StakeBelowMinimum)
2. Validate: `days >= 1 && days <= 5555` (error: InvalidStakeDuration)

5. Calculate end_slot: `clock.slot + (days as u64) * global_state.slots_per_day` (checked arithmetic)
6. Initialize StakeAccount fields: user, stake_id (from global_state.total_stakes_created), staked_amount, t_shares, start_slot, end_slot, stake_days, reward_debt (t_shares * share_rate, checked), is_active=true, bump
7. Update GlobalState: increment total_stakes_created, add amount to total_tokens_staked, add t_shares to total_shares (all checked)
8. **Burn tokens from user** using `anchor_spl::token_2022::burn()` CPI -- tokens are burned (locked in protocol) on stake creation, minted back on unstake. This is the burn-and-mint model from research.
9. Emit `StakeCreated` event with slot, user, stake_id, amount, t_shares, days, share_rate

**instructions/crank_distribution.rs** -- Create the permissionless crank instruction:

**Account struct `CrankDistribution`:**
- `cranker`: Signer (anyone can call -- permissionless)
- `global_state`: Account<GlobalState>, mut, seeds=[GLOBAL_STATE_SEED], bump=global_state.bump
- `mint`: InterfaceAccount<Mint>, mut, seeds=[MINT_SEED], bump
- `mint_authority`: UncheckedAccount, CHECK: PDA mint authority, seeds=[MINT_AUTHORITY_SEED], bump=global_state.mint_authority_bump
- `token_program`: Program<Token2022>

Note: This instruction does NOT need a treasury token account. The inflation is accounted for by increasing the share_rate. Tokens are only minted when users claim rewards (in Plan 02-03). The share_rate increase conceptually "distributes" inflation -- actual minting is deferred to claim time. This is more gas-efficient and follows the lazy distribution pattern from research.

**Wait -- reconsider.** The research shows two approaches:
1. Mint inflation on crank, store in treasury, users claim from treasury
2. Don't mint on crank, just update share_rate, mint on claim

Approach 2 is simpler (no treasury account needed) and follows the lazy pattern. But approach 1 is what HEX does (pool of rewards). For MVP, use approach 2: **crank only updates share_rate and current_day, no minting on crank.** Minting happens on claim and unstake.

**Handler `crank_distribution(ctx)`:**
1. Get clock: `Clock::get()?`
2. Calculate current_day: `get_current_day(global_state.init_slot, clock.slot, global_state.slots_per_day)?`
3. Require: `current_day > global_state.current_day` (error: AlreadyDistributedToday)
4. Calculate days elapsed: `current_day - global_state.current_day`
5. If total_shares == 0: just update current_day and return (no distribution if no stakes)
6. Calculate daily inflation amount:
   - Get total supply from mint account: `ctx.accounts.mint.supply`
   - `annual_inflation = supply * annual_inflation_bp / 100_000_000` (basis points with extra precision)
   - `daily_inflation = annual_inflation / 365 * days_elapsed` (checked arithmetic)
7. Update share_rate: `share_rate += daily_inflation * PRECISION / total_shares` (checked)
8. Update current_day: `global_state.current_day = current_day`
9. Emit `InflationDistributed` event

**IMPORTANT:** The crank does NOT mint tokens. It only updates share_rate. Tokens are minted lazily when users call claim_rewards or unstake. This avoids needing a treasury token account and simplifies the instruction.
