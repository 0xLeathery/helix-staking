## Decisions Made

**Math implementation pattern:**
- Used inline u128 casts in mul_div helpers instead of separate SafeMath module per anchor-expert recommendation (only 3 functions needed it, no need for module overhead)

**Penalty rounding direction:**
- Penalties round UP to favor protocol (user receives less) using mul_div_up helper per tokenomics-expert recommendation

**Late penalty timing:**
- Replaced LATE_PENALTY_BPS_PER_DAY constant with formula `penalty_bps = penalty_days * BPS_SCALER / LATE_PENALTY_WINDOW_DAYS` to hit exactly 100% at day 365 per tokenomics-expert recommendation

**Penalty redistribution:**
- Penalties increase share_rate for remaining stakers instead of being burned, redistributing value proportionally per tokenomics-expert recommendation
- Implementation order: calculate penalty → mark inactive → decrement shares → redistribute penalty → mint tokens
- Redistribution happens AFTER removing unstaker's shares so they don't benefit from their own penalty

**Admin mint security:**
- Used reserved[0] and reserved[1] fields for total_admin_minted and max_admin_mint to avoid account reallocation per security-expert recommendation
- Added AdminMinted event for transparency per security-expert recommendation
- Cap enforcement happens before minting to prevent overruns

**Indexer support:**
- Added days_elapsed to InflationDistributed event for gap detection per indexer-expert recommendation
