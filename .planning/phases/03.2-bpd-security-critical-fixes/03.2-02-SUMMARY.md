## Critical Vulnerabilities Fixed

### CRIT-1: First-Batch-Drains-Pool Attack
**Problem:** Per-batch rate calculation allowed first batch to consume disproportionate share of pool.

**Solution:**
- Added `bpd_calculation_complete` constraint to trigger_big_pay_day accounts struct
- Read pre-calculated `helix_per_share_day` from ClaimConfig instead of calculating per-batch
- Removed first-batch unclaimed calculation (now in finalize_bpd_calculation)
- Removed per-batch rate calculation logic
- Removed bpd_total_share_days accumulation in trigger (done in finalize)

**Verification:** Test "ensures cross-batch rate fairness" proves all batches use same rate.

### CRIT-2: Duplicate-Bonus-Drain Attack
**Problem:** Same stake could receive BPD multiple times within or across batches.

**Solution:**
- Check `bpd_claim_period_id` before distributing to each stake
- Set `bpd_claim_period_id = claim_period_id` after distributing bonus
- Prevents same stake from receiving BPD more than once per claim period

**Verification:** Tests "prevents same stake from receiving BPD multiple times across batches" and "prevents duplicate stakes within same batch" verify the fix.
