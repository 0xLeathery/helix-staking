**Accounts struct `FinalizeBpdCalculation<'info>`:**
- `caller: Signer<'info>` (permissionless, anyone can call)
- `global_state: Account<'info, GlobalState>` with seeds `[GLOBAL_STATE_SEED]`, bump = `global_state.bump`
- `claim_config: Account<'info, ClaimConfig>` with `mut`, seeds `[CLAIM_CONFIG_SEED]`, bump = `claim_config.bump`, constraints:
  - `claim_config.claim_period_started @ HelixError::ClaimPeriodNotStarted`
  - `!claim_config.bpd_calculation_complete @ HelixError::BpdCalculationAlreadyComplete`
  - `!claim_config.big_pay_day_complete @ HelixError::BigPayDayAlreadyTriggered`
- Remaining accounts: StakeAccounts to scan (read-only scan)

**Function `finalize_bpd_calculation<'info>`:**

The function signature should use the same lifetime pattern as trigger_big_pay_day:
`pub fn finalize_bpd_calculation<'info>(ctx: Context<'_, '_, 'info, 'info, FinalizeBpdCalculation<'info>>) -> Result<()>`

Logic:
1. Get clock, verify `clock.slot > claim_config.end_slot` (claim period ended) -- error `BigPayDayNotAvailable` if not
2. Determine if first batch: `claim_config.bpd_remaining_unclaimed == 0 && claim_config.bpd_total_share_days == 0`
3. If first batch, calculate unclaimed = `total_claimable - total_claimed`, store in `bpd_remaining_unclaimed`
4. If unclaimed == 0: set `bpd_calculation_complete = true`, set `bpd_helix_per_share_day = 0`, return Ok
5. Iterate `ctx.remaining_accounts` (max `MAX_STAKES_PER_FINALIZE`):
   - Same validation as trigger_big_pay_day: ownership check (`account_info.owner != &crate::id()`), size check (`data.len() < StakeAccount::LEN`), deserialization, PDA verification
   - Same eligibility checks: `is_active`, `start_slot >= claim_config.start_slot`, `start_slot <= claim_config.end_slot`
   - Calculate days_staked and share_days exactly as trigger_big_pay_day does (lines 143-156 of current code)
   - Accumulate `batch_share_days: u128`
6. Add batch_share_days to `claim_config.bpd_total_share_days`
7. Check if this is the last batch: `ctx.remaining_accounts.len() < MAX_STAKES_PER_FINALIZE`
8. If last batch AND `bpd_total_share_days > 0`:
   - Calculate: `bpd_helix_per_share_day = (unclaimed_amount as u128).checked_mul(PRECISION as u128)?.checked_div(bpd_total_share_days)?`
   - Store in `claim_config.bpd_helix_per_share_day`
   - Set `claim_config.bpd_calculation_complete = true`
9. If last batch AND `bpd_total_share_days == 0`:
   - No eligible stakes found at all. Reset pagination state (`bpd_remaining_unclaimed = 0`, `bpd_total_share_days = 0`) so it can be retried. Do NOT set calculation_complete. (Same "keep pool pending" pattern as current trigger_big_pay_day for empty first batch.)
10. Return Ok(())

CRITICAL: This instruction ONLY reads stake data for share-days accumulation. It does NOT write to any StakeAccount. It does NOT distribute anything. It ONLY updates ClaimConfig.
