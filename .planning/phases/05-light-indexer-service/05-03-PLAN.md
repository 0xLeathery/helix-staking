<objective>
Build the read-only REST API that serves historical indexed data to the frontend dashboard and future analytics pages.

Purpose: The dashboard (Phase 4) and analytics (Phase 6) need historical data that can't be read from on-chain state alone (e.g., T-share price over time, daily distribution history). This API turns the indexed Postgres data into consumable endpoints.
Output: A Fastify server at `services/indexer/src/api/index.ts` with health check and 4 data endpoints.
</objective>

  <name>Task 1: Create Fastify server with health check and aggregate stats endpoints</name>
  <files>
    services/indexer/src/api/index.ts

  <action>
    1. **src/api/index.ts**: Fastify server entrypoint.
       - Import and validate env at startup (fail fast).
       - Create Fastify instance with `logger: false` (use Pino separately for control).
       - Register `@fastify/cors` with `origin: env.FRONTEND_URL`, `credentials: true`.
       - Register all route plugins: healthRoutes, statsRoutes, distributionRoutes, stakeRoutes, claimRoutes.
       - Set global error handler:
         ```typescript
         fastify.setErrorHandler((error, request, reply) => {
           logger.error({ error: error.message, url: request.url }, 'Request error');
           reply.status(error.statusCode || 500).send({
             error: error.name || 'Internal Server Error',
             message: error.message,
             statusCode: error.statusCode || 500,
           });
         });
         ```
       - Start server on `env.PORT` (default 3001), host `0.0.0.0`.
       - Register SIGTERM/SIGINT handlers for graceful shutdown: `await fastify.close()`, then `pool.end()`, then `process.exit(0)`.
       - Log startup: port, environment, frontend URL.

    2. **src/api/routes/health.ts**: Health check endpoint.
       - Export `healthRoutes` as a Fastify plugin function.
       - `GET /health`:
         - Check database connectivity: `await db.execute(sql\`SELECT 1\`)` (import `sql` from drizzle-orm).
         - Query checkpoint table for last indexed slot.
         - Query RPC for current finalized slot via `rpc.getSlot('finalized')`.
         - Calculate lag = currentSlot - lastIndexedSlot.
         - If database unreachable: return 503 `{ status: 'unhealthy', error: '<message>' }`.
         - If lag > 1000 slots (~7 minutes): return 503 `{ status: 'degraded', lag, message: 'Indexer behind' }`.
         - Otherwise: return 200 `{ status: 'healthy', lag, lastSlot, currentSlot, uptime: process.uptime(), timestamp: new Date().toISOString() }`.
       - Create RPC client inside the plugin using env.RPC_URL (or accept it via plugin options).

    3. **src/api/routes/stats.ts**: Aggregate statistics endpoint.
       - Export `statsRoutes` as a Fastify plugin function.
       - `GET /api/stats`:
         - Query counts and aggregates from indexed data:
           - `totalStakes`: COUNT from stake_created_events
           - `totalUnstakes`: COUNT from stake_ended_events
           - `totalDistributions`: COUNT from inflation_distributed_events
           - `latestDistribution`: Most recent row from inflation_distributed_events (ORDER BY day DESC LIMIT 1) -- provides currentDay, currentShareRate, totalShares, lastDistributionAmount
           - `totalRewardsClaimed`: COUNT from rewards_claimed_events
           - `totalTokensClaimed`: COUNT from tokens_claimed_events
         - Return JSON:
           ```json
           {
             "totalStakes": 1234,
             "totalUnstakes": 56,
             "totalDistributions": 100,
             "totalRewardsClaimed": 789,
             "totalTokensClaimed": 200,
             "currentDay": 100,
             "currentShareRate": "1000000000",
             "totalShares": "50000000000",
             "lastDistributionAmount": "1000000",
             "lastUpdated": "2026-02-08T..."
           }
           ```
         - If no distributions exist yet, return zeros/nulls for distribution-related fields.
         - Use Drizzle's `count()` and `desc()` functions.
  </action>

<task type="auto">
  <name>Task 2: Create distribution history, stake history, and claim history endpoints</name>
  <files>
    services/indexer/src/api/routes/distributions.ts

  <action>
    1. **Shared pagination pattern** (use in all three routes):
       - Accept query params: `page` (default 1), `limit` (default 50, max 200)
       - Calculate offset: `(page - 1) * limit`
       - Return response shape:
         ```json
         {
           "data": [...],
           "pagination": {
             "page": 1,
             "limit": 50,
             "total": 1234,
             "totalPages": 25
           }
         }
         ```
       - Use Zod schemas to validate/coerce query params (page and limit as positive integers).

    2. **src/api/routes/distributions.ts**: Daily distribution history.
       - Export `distributionRoutes` as Fastify plugin.
       - `GET /api/distributions`:
         - Query `inflation_distributed_events` table.
         - Order by `day` DESC (newest first).
         - Apply pagination.
         - Return fields: `day`, `daysElapsed`, `amount`, `newShareRate`, `totalShares`, `slot`, `signature`, `createdAt`.
       - `GET /api/distributions/chart`:
         - Optimized endpoint for charting: returns all distributions with only `day`, `newShareRate`, `amount`, `totalShares`.
         - Order by `day` ASC (oldest first, for time-series charts).
         - No pagination (return all records -- expected to be <5000 rows for years of daily distributions).
         - Phase 6 (Analytics) will consume this endpoint for T-share price and payout charts.

    3. **src/api/routes/stakes.ts**: Stake event history.
       - Export `stakeRoutes` as Fastify plugin.
       - `GET /api/stakes`:
         - Query `stake_created_events` table.
         - Optional query param: `user` (filter by user address, case-sensitive).
         - Order by `slot` DESC.
         - Apply pagination.
         - Return fields: `user`, `stakeId`, `amount`, `tShares`, `days`, `shareRate`, `slot`, `signature`, `createdAt`.
       - `GET /api/unstakes`:
         - Query `stake_ended_events` table.
         - Optional query param: `user` (filter by user address).
         - Order by `slot` DESC.
         - Apply pagination.
         - Return fields: `user`, `stakeId`, `originalAmount`, `returnAmount`, `penaltyAmount`, `penaltyType`, `rewardsClaimed`, `slot`, `signature`, `createdAt`.

    4. **src/api/routes/claims.ts**: Free claim and reward claim history.
       - Export `claimRoutes` as Fastify plugin.
       - `GET /api/claims/tokens`:
         - Query `tokens_claimed_events` table.
         - Optional query param: `claimer` (filter by claimer address).
         - Order by `slot` DESC.
         - Apply pagination.
         - Return fields: `claimer`, `snapshotWallet`, `claimPeriodId`, `snapshotBalance`, `baseAmount`, `bonusBps`, `totalAmount`, `immediateAmount`, `vestingAmount`, `slot`, `signature`, `createdAt`.
       - `GET /api/claims/rewards`:
         - Query `rewards_claimed_events` table.
         - Optional query param: `user` (filter by user address).
         - Order by `slot` DESC.
         - Apply pagination.
         - Return fields: `user`, `stakeId`, `amount`, `slot`, `signature`, `createdAt`.
       - `GET /api/claims/bpd`:
         - Query `big_pay_day_distributed_events` table.
         - Order by `slot` DESC.
         - No user filter (BPD events are global).
         - Apply pagination.
         - Return fields: `claimPeriodId`, `totalUnclaimed`, `totalEligibleShareDays`, `helixPerShareDay`, `eligibleStakers`, `slot`, `signature`, `createdAt`.

    **Important implementation details:**
    - Use `eq()` from drizzle-orm for WHERE clauses (e.g., `eq(table.user, userParam)`).
    - Use `count()` for total count queries (for pagination metadata).
    - All responses should be JSON with consistent field naming (camelCase matching the schema column names).
    - If `user`/`claimer` filter is provided but no results found, return empty `data: []` with `total: 0` (not 404).
  </action>
