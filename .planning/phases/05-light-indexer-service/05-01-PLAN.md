  <action>
    1. **src/types/events.ts**: Define TypeScript interfaces mirroring all 11 program events from `programs/helix-staking/src/events.rs`:
       - `ProtocolInitialized`: slot, globalState, mint, mintAuthority, authority, annualInflationBp, minStakeAmount, startingShareRate, slotsPerDay
       - `StakeCreated`: slot, user, stakeId, amount, tShares, days, shareRate
       - `StakeEnded`: slot, user, stakeId, originalAmount, returnAmount, penaltyAmount, penaltyType (0=None, 1=Early, 2=Late), rewardsClaimed
       - `RewardsClaimed`: slot, user, stakeId, amount
       - `InflationDistributed`: slot, day, daysElapsed, amount, newShareRate, totalShares
       - `AdminMinted`: slot, authority, recipient, amount
       - `ClaimPeriodStarted`: slot, timestamp, claimPeriodId, merkleRoot, totalClaimable, totalEligible, claimDeadlineSlot
       - `TokensClaimed`: slot, timestamp, claimer, snapshotWallet, claimPeriodId, snapshotBalance, baseAmount, bonusBps, daysElapsed, totalAmount, immediateAmount, vestingAmount, vestingEndSlot
       - `VestedTokensWithdrawn`: slot, timestamp, claimer, amount, totalVested, totalWithdrawn, remaining
       - `ClaimPeriodEnded`: slot, timestamp, claimPeriodId, totalClaimed, claimsCount, unclaimedAmount
       - `BigPayDayDistributed`: slot, timestamp, claimPeriodId, totalUnclaimed, totalEligibleShareDays, helixPerShareDay, eligibleStakers

       Use camelCase field names (Anchor JS SDK convention). All Pubkey fields as `string`, all u64/u128 as `string` (BN.toString()), u8/u16/u32 as `number`, `[u8; 32]` as `number[]`.

       Also export a discriminated union type `IndexedEvent = { name: string; data: <event-type> }` and an `EVENT_NAMES` array of all 11 event name strings.

    2. **src/db/schema.ts**: Define Drizzle pgTable schemas for all event storage plus operational tables:

       **Shared columns pattern**: Every event table has `id` (serial, PK), `signature` (text, unique, not null), `slot` (bigint mode number, not null), `createdAt` (timestamp, defaultNow).

       Event tables (one per event type):
       - `protocol_initialized_events`: globalState, mint, mintAuthority, authority (all text), annualInflationBp, minStakeAmount, startingShareRate, slotsPerDay (all bigint mode number)
       - `stake_created_events`: user (text, indexed), stakeId (bigint), amount (bigint), tShares (bigint), days (integer), shareRate (bigint). Index on user + slot.
       - `stake_ended_events`: user (text, indexed), stakeId (bigint), originalAmount (bigint), returnAmount (bigint), penaltyAmount (bigint), penaltyType (integer), rewardsClaimed (bigint). Index on user.
       - `rewards_claimed_events`: user (text, indexed), stakeId (bigint), amount (bigint). Index on user.
       - `inflation_distributed_events`: day (bigint, indexed), daysElapsed (bigint), amount (bigint), newShareRate (bigint), totalShares (bigint). Index on day.
       - `admin_minted_events`: authority (text), recipient (text), amount (bigint).
       - `claim_period_started_events`: timestamp (bigint), claimPeriodId (integer), merkleRoot (text), totalClaimable (bigint), totalEligible (integer), claimDeadlineSlot (bigint).
       - `tokens_claimed_events`: timestamp (bigint), claimer (text, indexed), snapshotWallet (text), claimPeriodId (integer), snapshotBalance (bigint), baseAmount (bigint), bonusBps (integer), daysElapsed (integer), totalAmount (bigint), immediateAmount (bigint), vestingAmount (bigint), vestingEndSlot (bigint).
       - `vested_tokens_withdrawn_events`: timestamp (bigint), claimer (text, indexed), amount (bigint), totalVested (bigint), totalWithdrawn (bigint), remaining (bigint).
       - `claim_period_ended_events`: timestamp (bigint), claimPeriodId (integer), totalClaimed (bigint), claimsCount (integer), unclaimedAmount (bigint).
       - `big_pay_day_distributed_events`: timestamp (bigint), claimPeriodId (integer), totalUnclaimed (bigint), totalEligibleShareDays (bigint), helixPerShareDay (bigint), eligibleStakers (integer).

       **Operational tables**:
       - `checkpoints`: id (serial PK), programId (text, unique), lastSignature (text), lastSlot (bigint mode number), processedCount (bigint mode number, default 0), updatedAt (timestamp, defaultNow).

       Use Drizzle's `index()` for indexed columns. Use `bigint('col', { mode: 'number' })` for numeric fields. Store Pubkeys and large numbers as text to avoid JavaScript number precision issues -- EXCEPT for slot/day/counts where number mode is fine (they fit in JS safe integers).

       IMPORTANT: For `amount`, `tShares`, `shareRate`, and other token-value fields that could exceed JS safe integers, use `text` type (store as string). For `slot`, `day`, `stakeId`, `daysElapsed`, and counter fields that stay within safe integer range, use `bigint` with `mode: 'number'`.

    3. **src/db/client.ts**: Initialize Drizzle with Neon serverless pool.
       ```typescript
       import { Pool } from '@neondatabase/serverless';
       import { drizzle } from 'drizzle-orm/neon-serverless';
       import * as schema from './schema.js';
       ```
       Create pool with `connectionString` from env, `max: 10`. Export `db` (drizzle instance) and `pool` (for graceful shutdown).

    4. **src/lib/rpc.ts**: Create retryable RPC client wrapper.
       - Export `createRpcClient(endpoint: string)` that returns an object with:
         - `getSignaturesForAddress(address, options)` -- wraps Connection method with p-retry (5 retries, factor 2, minTimeout 1000, maxTimeout 60000, log failures via logger)
         - `getParsedTransaction(signature, options)` -- same retry pattern
         - `getSlot(commitment)` -- same retry pattern
         - `connection` -- raw Connection for direct access if needed
       - Use `confirmed` commitment by default.

    5. **src/lib/anchor.ts**: Create Anchor event decoder.
       - Import IDL from `../../../target/idl/helix_staking.json` (with assert { type: 'json' } or using readFileSync + JSON.parse for ESM compatibility)
       - Initialize `BorshCoder` and `EventParser` from `@coral-xyz/anchor`
       - Export `parseEventsFromLogs(logs: string[]): Array<{ name: string; data: any }>` that uses eventParser.parseLogs()
       - Export `programId` from the IDL
       - Handle the IDL import carefully -- if JSON import assertion causes issues, use `fs.readFileSync` + `JSON.parse` with `path.resolve` relative to project root. Accept IDL path as optional env var `IDL_PATH` with default `../../target/idl/helix_staking.json`.

    Run `npx tsc --noEmit --skipLibCheck` from services/indexer to verify everything compiles.
  </action>
