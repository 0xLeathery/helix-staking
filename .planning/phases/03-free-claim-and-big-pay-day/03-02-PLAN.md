**claim_rewards.rs** -- Update to handle lazy migration and BPD bonus payout.

use anchor_lang::system_program::{self, Transfer};

2. Update ClaimRewards struct to support realloc:

Add `realloc` constraint to stake_account:
```rust
    #[account(
        mut,
        seeds = [STAKE_SEED, user.key().as_ref(), &stake_account.stake_id.to_le_bytes()],
        bump = stake_account.bump,
        constraint = stake_account.user == user.key() @ HelixError::UnauthorizedStakeAccess,
        constraint = stake_account.is_active @ HelixError::StakeNotActive,
        realloc = StakeAccount::LEN,
        realloc::payer = user,
        realloc::zero = true,
    )]
    pub stake_account: Account<'info, StakeAccount>,
```

Note: `realloc::zero = true` zeroes new bytes after expansion, ensuring BPD fields are safely initialized. Anchor handles rent adjustment automatically.

3. Update `claim_rewards` function to include BPD bonus:

After calculating pending_rewards, add BPD bonus:
```rust
    // Include BPD bonus if pending
    let bpd_bonus = stake.bpd_bonus_pending;
    let total_rewards = pending_rewards
        .checked_add(bpd_bonus)
        .ok_or(HelixError::Overflow)?;

    // Require non-zero total rewards
    require!(total_rewards > 0, HelixError::NoRewardsToClaim);
```

After updating reward_debt, clear BPD bonus:
```rust
    // Clear BPD bonus after claiming
    if bpd_bonus > 0 {
        stake_mut.bpd_bonus_pending = 0;
    }
```

Update the mint amount to use `total_rewards` instead of `pending_rewards`.

Update the event to include BPD info (or emit separate event if needed):
```rust
    emit!(RewardsClaimed {
        slot: clock.slot,
        user,
        stake_id,
        amount: total_rewards,  // Includes BPD bonus
    });
```

4. Handle migration for old accounts:

The `realloc` constraint with `realloc::zero = true` automatically expands the account and zeroes new bytes. No explicit initialization needed - the new BPD fields (bpd_bonus_pending, bpd_eligible, claim_period_start_slot) will be zero after migration, which is the correct default state.
