  <action>
    **1. useClaimConfig hook (`lib/hooks/useClaimConfig.ts`):**
    - React Query hook with queryKey: ["claimConfig"]
    - queryFn: fetch ClaimConfig PDA using deriveClaimConfig()
    - staleTime: 60_000 (claim config changes rarely)
    - Returns: claim period info, merkle root, BPD status fields (bpd_helix_per_share_day, bpd_calculation_complete, etc.)
    - Handle account-not-found gracefully: return null (claim period not initialized yet)

    **2. useFreeClaim mutation hook (`lib/hooks/useFreeClaim.ts`):**
    - Input: `{ snapshotWallet: PublicKey, amount: BN, proof: Buffer[], signature: Uint8Array, message: Uint8Array }`
    - MutationFn:
      1. Verify wallet connected
      2. Derive accounts: claimConfigPda, claimStatusPda (using merkle root prefix + snapshotWallet), mintPda, mintAuthorityPda, globalStatePda, user's Token-2022 ATA
      3. CRITICAL: Build Ed25519 verify instruction that MUST immediately precede free_claim instruction in same transaction (per on-chain requirement + CONTEXT.md)
        - Use Ed25519Program.createInstructionWithPublicKey() or manual construction (check test suite pattern)
        - The Ed25519 instruction verifies the signature over the message
      4. Build free_claim instruction via program.methods.freeClaim(amount, proof).accounts({...}).instruction()
      5. Create transaction with BOTH instructions: [ed25519Instruction, freeClaimInstruction]
      6. SIMULATE before sending
      7. Send and confirm
    - Handle errors:
      - "AlreadyClaimed" -> "You have already claimed your HELIX tokens"
      - "InvalidMerkleProof" -> "Invalid claim proof. Please verify your eligibility."
      - "ClaimPeriodNotActive" -> "The claim period has not started yet"
    - onSuccess: invalidate ["tokenBalance", publicKey], ["claimStatus"]

    **3. useWithdrawVested mutation hook (`lib/hooks/useWithdrawVested.ts`):**
    - Input: `{ claimStatusPublicKey: PublicKey }`
    - Derive accounts: claimConfigPda, claimStatusPda, mintPda, mintAuthorityPda, user's Token-2022 ATA
    - Build withdraw_vested transaction
    - SIMULATE before sending
    - Handle "NothingToWithdraw" error
    - onSuccess: invalidate ["claimStatus"], ["tokenBalance", publicKey]

    **4. Eligibility check (`components/claim/eligibility-check.tsx`):**
    - "use client" component
    - Checks if claim period is active (from useClaimConfig)
    - Shows:
      - Claim period status: "Active" / "Not Started" / "Ended"
      - If active: show claim form
      - If not started: show countdown or "Coming Soon"
      - If ended: show "Claim period has ended" + link to BPD status
    - Check if user has already claimed by querying ClaimStatus PDA:
      - Use React Query hook to fetch ClaimStatus account
      - If account exists with withdrawn_amount > 0: show "Already Claimed" status
    - Note: For MVP, the Merkle proof data (JSON) is loaded from a static CDN URL. For now, use a placeholder URL that the user will configure. The claim form will need: wallet address lookup in the proof data, amount, proof array.

    **5. Claim form (`components/claim/claim-form.tsx`):**
    - "use client" component
    - Steps:
      1. Load Merkle proof data from static JSON (fetch from CDN URL stored in env var `NEXT_PUBLIC_MERKLE_DATA_URL`)
      2. Look up connected wallet in proof data
      3. If found: show claimable amount and "Claim HELIX" button
      4. If not found: show "Your wallet is not eligible for the free claim"
    - Speed bonus display:
      - Week 1: "+20% Speed Bonus"
      - Weeks 2-4: "+10% Speed Bonus"
      - After week 4: "No Speed Bonus"
      - Show which tier the user qualifies for based on current time vs claim period start
    - "Claim HELIX" button triggers useFreeClaim mutation
    - Loading/error/success states

    **6. Vesting status (`components/claim/vesting-status.tsx`):**
    - "use client" component
    - Shows ONLY if user has an active claim (ClaimStatus account exists)
    - Display:
      - Total Claimed: X HELIX
      - Immediately Available: 10% of total (per 30-day linear vesting with 10% immediate)
      - Vesting Progress: visual progress bar (0-100% over 30 days)
      - Already Withdrawn: X HELIX
      - Available to Withdraw: calculated based on time elapsed in vesting period
      - "Withdraw Available" button (calls useWithdrawVested)
    - Vesting math: 10% immediate + 90% * (days_elapsed / 30)
    - Show "Fully Vested" state when 30 days have passed

    **7. Claim page (`app/dashboard/claim/page.tsx`):**
    - Compose: EligibilityCheck at top, then ClaimForm (if eligible), then VestingStatus (if claimed)
    - Page title: "Free Claim"
    - Educational tooltip on "Free Claim": "SOL holders from the snapshot can claim free HELIX tokens proportional to their balance"
