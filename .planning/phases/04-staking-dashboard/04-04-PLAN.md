  <files>

    app/web/lib/hooks/useCurrentSlot.ts

  <action>
    **1. useCurrentSlot hook (`lib/hooks/useCurrentSlot.ts`):**
    - React Query hook with queryKey: ["currentSlot"]
    - queryFn: connection.getSlot("confirmed")
    - staleTime: 5_000 (5s -- slots change every ~400ms but we don't need real-time)
    - refetchInterval: 10_000 (10s polling)
    - Returns current slot as number (convert to BN when needed for math)

    **2. useUnstake mutation hook (`lib/hooks/useUnstake.ts`):**
    - Uses useMutation from React Query
    - Input: `{ stakePublicKey: PublicKey }` (the specific stake account PDA)
    - MutationFn:
      1. Verify wallet connected
      2. Derive all required accounts: globalStatePda, mintPda, mintAuthorityPda, user's Token-2022 ATA
      3. Build transaction via program.methods.unstake().accounts({...}).transaction()
      4. Set recentBlockhash and feePayer
      5. SIMULATE transaction before sending (security requirement)
      6. If simulation fails, parse error for user-friendly message:
         - "UnstakeBlockedDuringBpd" -> "Unstaking is temporarily unavailable during Big Pay Day calculation"
         - Other errors -> generic "Transaction failed" with error details
      7. Send via wallet.sendTransaction
      8. Confirm (commitment: "confirmed")
      9. Return signature
    - onSuccess: invalidate ["stakes", publicKey], ["globalState"], ["tokenBalance", publicKey]
    - NOTE: On-chain unstake automatically claims pending rewards in same transaction (per Phase 2 decision). The UI should communicate this: "Ending your stake will also claim all pending rewards."

    **3. useClaimRewards mutation hook (`lib/hooks/useClaimRewards.ts`):**
    - Uses useMutation from React Query
    - Input: `{ stakePublicKey: PublicKey }`
    - MutationFn:
      1. Verify wallet connected
      2. Derive accounts: globalStatePda, mintPda, mintAuthorityPda, user's Token-2022 ATA
      3. Build transaction via program.methods.claimRewards().accounts({...}).transaction()
      4. SIMULATE before sending
      5. Send and confirm
      6. Return signature
    - Handle "NoRewardsToClaim" error gracefully: show toast "No rewards available to claim yet"
    - onSuccess: invalidate ["stakes", publicKey], ["tokenBalance", publicKey]
  </action>

    Verify both hooks include simulateTransaction call.
    Verify useUnstake handles "UnstakeBlockedDuringBpd" error.
    Verify useClaimRewards handles "NoRewardsToClaim" error.
  </verify>
  <done>

  </done>

  <action>
    **1. Penalty calculator (`components/stake/penalty-calculator.tsx`):**
    - "use client" component receiving stake account data and current slot
    - Determines stake state:
      - **Early**: currentSlot < endSlot -> use calculateEarlyPenalty
      - **On-time (grace period)**: endSlot <= currentSlot <= endSlot + GRACE_PERIOD_DAYS * SLOTS_PER_DAY -> penalty = 0
      - **Late**: currentSlot > graceEndSlot -> use calculateLatePenalty
    - Displays:
      - Status label: "Early Unstake" (yellow), "End Stake" (green), "End Stake (Late)" (red) per user terminology
      - Visual comparison bar: "What you staked" (full bar) vs "What you'd receive" (proportional bar). Use green for receive amount, red for penalty amount. Side-by-side or stacked bars.
      - Staked Amount: X HELIX
      - Penalty Amount: X HELIX (Y%)
      - You Receive: X HELIX (staked - penalty + pending rewards + bpd_bonus_pending)
      - Pending Rewards: X HELIX (will be claimed with unstake)
      - BPD Bonus: X HELIX (if applicable)
    - Color coding per user decision:
      - Green: matured/grace period (penalty = 0)
      - Yellow: early with moderate penalty (< 75%)
      - Red: heavy penalty (>= 75%) or late unstake
    - For late unstake: show urgency indicator -- "X days until total loss" with days remaining before 100% penalty

    **2. Unstake confirmation (`components/stake/unstake-confirmation.tsx`):**
    - Dialog component (uses shadcn/ui Dialog)
    - Shows penalty summary (abbreviated version of penalty calculator)
    - Mandatory checkbox: "I understand I will lose X HELIX (Y%)" -- REQUIRED before confirming (per user decision: no way to skip)
    - For on-time unstake (0% penalty), checkbox text: "I understand this will end my stake and claim all rewards"
    - "Cancel" and "End Stake" buttons
    - "End Stake" button disabled until checkbox checked AND mutation not pending
    - On confirm: call useUnstake.mutateAsync({ stakePublicKey })
    - Show loading spinner during transaction
    - On success: close dialog, show success toast "Stake ended successfully! Tokens returned to your wallet.", navigate to /dashboard
    - On error: show error toast, keep dialog open

    **3. Stake detail page (`app/dashboard/stakes/[stakeId]/page.tsx`):**
    - "use client" page with dynamic route parameter [stakeId]
    - The stakeId is the base58 public key of the StakeAccount PDA
    - Fetch individual stake: use program.account.stakeAccount.fetch(new PublicKey(stakeId))
      - Or find in useStakes results to avoid extra RPC call
    - Full stake details card:
      - Staked Amount (formatHelix)
      - T-Shares (formatTShares)
      - Duration Bonus / Size Bonus (calculated and displayed)
      - Start Date (derived from start_slot and slot timing)
      - End Date (derived from start_slot + stake_days * SLOTS_PER_DAY)
      - Days Remaining (or "Matured X days ago")
      - Pending Rewards (calculatePendingRewards)
      - BPD Bonus (bpd_bonus_pending if > 0)
    - Penalty Calculator component (always visible below details)
    - Action buttons:
      - "End Stake" / "Early Unstake" / "End Stake (Late)" (label per state, per user terminology)
        - Opens UnstakeConfirmation dialog
        - DISABLED if BPD window active (globalState.reserved[0] !== 0)
        - When disabled, show yellow banner: "Unstaking is temporarily unavailable during Big Pay Day calculation. This ensures accurate bonus distribution. Please try again in a few minutes." (per Pitfall 3)
      - "Claim Rewards" button
        - Calls useClaimRewards.mutateAsync
        - Disabled if pending rewards = 0
        - Show toast on success/error
        - Note: "Ending your stake will also claim all rewards" helper text near this button
    - "Back to Dashboard" link
    - Loading: Skeleton for full page
    - 404: "Stake not found" if fetch fails
