  <action>
    **1. Zustand wizard store (`lib/store/ui-store.ts`):**
    - Create with Zustand v5 `create` function
    - State shape:
      ```
      step: 1 | 2 | 3 | "success"
      amount: string (user input, decimal format e.g. "100.5")
      days: number (1-5555)
      setStep, setAmount, setDays
      reset() - clears form and returns to step 1
      ```
    - Keep as UI-only state (no on-chain data per architecture decision)
    - Validate: amount must be string (parsed to BN only on submit), days must be integer 1-5555

    **2. Bonus preview component (`components/stake/bonus-preview.tsx`):**
    - "use client" component receiving `amount: BN` and `days: number` props
    - Uses calculateLpbBonus, calculateBpbBonus, calculateTShares from math.ts
    - Uses useGlobalState hook to get current shareRate
    - Displays:
      - Duration Bonus: X.XX% (formatted from LPB, label "Duration Bonus" per user terminology)
      - Size Bonus: X.XX% (formatted from BPB, label "Size Bonus" per user terminology)
      - Total Multiplier: X.XXx (1 + LPB/PRECISION + BPB/PRECISION)
      - Estimated T-Shares: formatted result
      - T-Share Price: current shareRate from globalState (label "T-Share Price" per user terminology)
    - Visual: card with each line item, highlight T-shares in accent color
    - Bonus curve visualization: simple horizontal progress bars showing where the user's amount/duration falls on the bonus curve (0-100% of max bonus)
    - Updates live as user changes amount/duration (no debounce needed -- BN math is fast)

    **3. Create stake mutation hook (`lib/hooks/useCreateStake.ts`):**
    - Uses useMutation from React Query
    - Input: `{ amount: BN, days: number }`
    - MutationFn:
      1. Verify wallet connected (throw if not)
      2. Fetch globalState to get totalStakesCreated
      3. Derive stakeAccountPda using user pubkey + totalStakesCreated
      4. Derive mintPda, globalStatePda
      5. Get user's Token-2022 ATA (using getAssociatedTokenAddress with TOKEN_2022_PROGRAM_ID -- per Pitfall 4)
      6. Build transaction via program.methods.createStake(amount, days).accounts({...}).transaction()
      7. Set recentBlockhash and feePayer
      8. SIMULATE transaction before sending (user security requirement)
      9. If simulation fails, throw with user-friendly error message
      10. Send via wallet.sendTransaction(tx, connection)
      11. Confirm transaction (commitment: "confirmed")
      12. Return signature
    - Handle Stake ID race condition (Pitfall 1):
      - Wrap in retry loop (max 3 retries)
      - Catch "already in use" errors
      - Re-fetch globalState and retry with updated totalStakesCreated
      - Wait 500ms between retries
    - onSuccess: invalidate ["stakes", publicKey], ["globalState"], ["tokenBalance", publicKey]
    - onError: console.error with full error, surface user-friendly message
    - Return full mutation object (mutateAsync, isPending, error, etc.)
  </action>
