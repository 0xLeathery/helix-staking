  <action>
    **1. useProgram hook (`lib/hooks/useProgram.ts`):**
    - "use client" hook
    - Uses `useConnection` and `useWallet` from wallet adapter
    - Returns memoized `Program<HelixStaking>` via `getProgram(connection, wallet)` from lib/solana/program.ts
    - When wallet is null/disconnected, pass dummy wallet for read-only access

    **2. useGlobalState hook (`lib/hooks/useGlobalState.ts`):**
    - React Query hook with queryKey: ["globalState"]
    - queryFn: fetch globalState PDA using program.account.globalState.fetch()
    - staleTime: 30_000 (30s per user decision)
    - refetchInterval: 60_000 (fallback polling per Pitfall 6)
    - refetchOnWindowFocus: true
    - WebSocket subscription via connection.onAccountChange on globalState PDA
    - On account change: invalidateQueries({ queryKey: ["globalState"] })
    - Cleanup: removeAccountChangeListener on unmount
    - Return the full React Query result (data, isLoading, error, etc.)

    **3. useStakes hook (`lib/hooks/useStakes.ts`):**
    - React Query hook with queryKey: ["stakes", publicKey?.toBase58()]
    - queryFn: use `program.account.stakeAccount.all([{ memcmp: { offset: 8, bytes: publicKey.toBase58() } }])` to filter by user (Pitfall 5)
    - enabled: !!publicKey (only fetch when wallet connected)
    - staleTime: 15_000 (15s per user decision)
    - refetchInterval: 30_000 (fallback polling)
    - refetchOnWindowFocus: true
    - WebSocket: subscribe to each stake account PDA for instant cache invalidation
    - Return accounts array (each has .publicKey and .account properties)

    **4. useTokenBalance hook (`lib/hooks/useTokenBalance.ts`):**
    - React Query hook with queryKey: ["tokenBalance", publicKey?.toBase58()]
    - queryFn: derive user's Token-2022 ATA (using getAssociatedTokenAddress with TOKEN_2022_PROGRAM_ID -- Pitfall 4), then fetch balance via connection.getTokenAccountBalance()
    - enabled: !!publicKey
    - staleTime: 10_000
    - Return balance as BN (token base units)

    **5. Math module (`lib/solana/math.ts`):**
    - Must match on-chain math.rs EXACTLY. Read `programs/helix-staking/src/instructions/math.rs` to verify formulas.
    - All functions use BN.js, never JS Number for amounts.
    - Export:
      - `calculateLpbBonus(stakeDays: BN): BN` - Duration bonus (0 to 2*PRECISION at 3641 days). Formula: (days-1) * 2 * PRECISION / LPB_MAX_DAYS. Return 0 for 0 days, 2*PRECISION for >= LPB_MAX_DAYS.
      - `calculateBpbBonus(stakedAmount: BN): BN` - Size bonus (0 to PRECISION at 150M threshold). Formula: (amount/10) * PRECISION / BPB_THRESHOLD. Cap at PRECISION.
      - `calculateTShares(stakedAmount: BN, stakeDays: BN, shareRate: BN): BN` - T-shares = amount * (PRECISION + lpb + bpb) / shareRate.
      - `calculateEarlyPenalty(stakedAmount: BN, startSlot: BN, currentSlot: BN, endSlot: BN): BN` - Early penalty with 50% minimum. Return 0 if currentSlot >= endSlot.
      - `calculateLatePenalty(stakedAmount: BN, endSlot: BN, currentSlot: BN, slotsPerDay: BN): BN` - Late penalty after grace period, linear to 100% over 351 days.
      - `calculatePendingRewards(tShares: BN, shareRate: BN, rewardDebt: BN): BN` - Accrued rewards = (tShares * shareRate / PRECISION) - rewardDebt.
    - Import constants from constants.ts.

    **6. Format utilities (`lib/utils/format.ts`):**
    - `formatHelix(amount: BN, showSymbol?: boolean): string` - Format token amount with 8 decimals. Example: BN(100_000_000) -> "1.00000000 HELIX". Trim trailing zeros for display but keep at least 2 decimal places.
    - `formatHelixCompact(amount: BN): string` - Compact format: 1.5M, 250K, etc. For large amounts in protocol stats.
    - `parseHelix(input: string): BN` - Parse user input "1.5" -> BN(150_000_000).
    - `formatBps(bps: number): string` - Basis points to percent: 5000 -> "50.00%".
    - `formatDays(days: number): string` - Human-readable: "1 day", "365 days", "3.5 years".
    - `formatTShares(amount: BN): string` - Format T-shares with precision display.
    - `truncateAddress(address: string): string` - "AbCd...xYzQ" (4 chars each end).
    - Use HELIX terminology per user decisions: "Duration Bonus" (not LPB), "Size Bonus" (not BPB), "T-Share Price" (not share_rate).
