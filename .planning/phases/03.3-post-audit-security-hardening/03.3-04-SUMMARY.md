## Lessons Learned

1. **Test runner environment matters** - Noble/hashes ESM export issue is a known problem with ts-mocha and newer Node versions
2. **Program rebuild required for IDL changes** - Always rebuild after adding new instructions before writing tests
3. **Security tests benefit from integration scenarios** - Single comprehensive integration test validates all security measures work together
4. **Counter-based logic is easier to test** - Counter verification (`bpd_stakes_distributed >= bpd_stakes_finalized`) is clearer than heuristic-based completion
