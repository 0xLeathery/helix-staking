## Decisions Made

**finalize_bpd_calculation changes:**
- Capture bpd_snapshot_slot on first batch (when bpd_remaining_unclaimed == 0 && bpd_total_share_days == 0 && bpd_snapshot_slot == 0) for consistent days_staked across all batches
- Set BPD window active on first batch to block unstake during distribution period
- Check bpd_finalize_period_id == claim_period_id early to skip already-counted stakes (duplicate prevention)
- Mark each eligible stake with bpd_finalize_period_id and write back to account
- Increment bpd_stakes_finalized counter per eligible stake
- Skip un-migrated stakes (data.len() < StakeAccount::LEN = 117 bytes)
- Remove last-batch detection and rate calculation (now done by seal_bpd_finalize instruction)
- Handle zero-eligible-stakes by setting rate=0 and bpd_calculation_complete=true (MED-4 fix)

**trigger_big_pay_day changes:**
- Load snapshot_slot from claim_config and use for days_staked calculation (consistent with finalize)
- Verify each stake has bpd_finalize_period_id == claim_period_id before distributing (CRIT-NEW-1)
- Use u64::try_from instead of 'as u64' for bonus cast to catch overflow (MED-1)
- Use checked_sub instead of saturating_sub for bpd_remaining_unclaimed to detect over-distribution (MED-3)
- Track batch_stakes_distributed (u32) and update claim_config.bpd_stakes_distributed
- Replace len < MAX heuristic with counter-based completion: bpd_stakes_distributed >= bpd_stakes_finalized
- Clear BPD window on completion (both rate=0 path and normal completion)
- Remove empty-batch last-batch heuristic (return early if eligible_stakes.is_empty())

**unstake changes:**
- Add BPD window guard: require!(!global_state.is_bpd_window_active(), HelixError::UnstakeBlockedDuringBpd)
- Include bpd_bonus_pending in total_mint_amount calculation
- Clear bpd_bonus_pending after including in payout
- Include BPD bonus in StakeEnded event rewards_claimed field
