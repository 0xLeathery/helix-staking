## Decisions Made

**State expansion strategy:**
- StakeAccount and ClaimConfig expanded with new fields required for Phase 3.3 security hardening
- Existing unused fields (bpd_eligible, claim_period_start_slot, authority) marked as DEPRECATED with comments explaining they're kept for layout compatibility
- PHASE3_LEN constant added to StakeAccount for detecting pre-3.3 accounts during migration

**BPD window flag implementation:**
- GlobalState.reserved[0] used for BPD window active flag to avoid schema reallocation
- Helper methods is_bpd_window_active() and set_bpd_window_active() encapsulate the logic

**Sealing instruction design:**
- seal_bpd_finalize is authority-gated (only protocol authority can call)
- Performs final validation before calculating BPD rate
- Requires claim period ended and finalize phase completed
- Locks the rate before distribution begins

**MED-5 fix rationale:**
- claim_period_id=0 would collide with default bpd_claim_period_id=0 in StakeAccount
- This collision would cause trigger_big_pay_day to skip ALL stakes (because none != current period)
- Validation added immediately in initialize_claim_period to reject claim_period_id=0
