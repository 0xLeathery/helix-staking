  <files>tests/bankrun/initialize.test.ts, tests/bankrun/createStake.test.ts, tests/bankrun/unstake.test.ts, tests/bankrun/claimRewards.test.ts, tests/bankrun/crankDistribution.test.ts, tests/bankrun/phase3/initializeClaim.test.ts, tests/bankrun/phase3/freeClaim.test.ts, tests/bankrun/phase3/withdrawVested.test.ts, tests/bankrun/phase3/migration.test.ts, tests/bankrun/phase3/triggerBpd.test.ts, tests/bankrun/phase3.3/securityHardening.test.ts</files>
  <action>
    1. In ALL test files, replace mocha/chai imports with vitest equivalents:
       - Replace `import { describe, it } from "mocha"` with `import { describe, it, expect } from "vitest"`
       - Replace `import { expect } from "chai"` with nothing (already imported from vitest)
       - Replace `import { before, beforeEach, after } from "mocha"` with `import { beforeAll, beforeEach, afterAll } from "vitest"` and rename `before()` calls to `beforeAll()`, `after()` to `afterAll()`
    2. Migrate chai assertion syntax to vitest:
       - `expect(x).to.equal(y)` -> `expect(x).toBe(y)` (for primitives)
       - `expect(x).to.deep.equal(y)` -> `expect(x).toEqual(y)` (for objects)
       - `expect(x).to.be.true` -> `expect(x).toBe(true)`
       - `expect(x).to.be.false` -> `expect(x).toBe(false)`
       - `expect(x).to.be.null` -> `expect(x).toBeNull()`
       - `expect(x).to.be.undefined` -> `expect(x).toBeUndefined()`
       - `expect(x).to.be.greaterThan(y)` -> `expect(x).toBeGreaterThan(y)`
       - `expect(x).to.be.lessThan(y)` -> `expect(x).toBeLessThan(y)`
       - `expect(x).to.be.at.least(y)` -> `expect(x).toBeGreaterThanOrEqual(y)`
       - `expect(x).to.exist` -> `expect(x).toBeDefined()`
       - `expect(x).to.not.exist` -> `expect(x).toBeUndefined()` or `expect(x).toBeNull()`
       - `expect(fn).to.throw()` -> wrap in `expect(() => fn()).toThrow()` or for async: `await expect(fn()).rejects.toThrow()`
       - `expect(x).to.include(y)` -> `expect(x).toContain(y)`
    3. For error assertion tests that use try/catch pattern:
       - If test catches errors from Anchor: keep existing try/catch pattern but use vitest assertions
       - `expect(error).to.exist` -> `expect(error).toBeDefined()`
       - The key pattern is: `try { await tx; expect.fail("should throw"); } catch (e) { expect(e).toBeDefined(); }`
       - vitest equivalent: `try { await tx; expect(true).toBe(false); } catch (e) { expect(e).toBeDefined(); }`
       - Or better: `await expect(asyncFn()).rejects.toThrow()`
    4. Build the program first: `anchor build`
    5. Run all tests: `npm run test:bankrun`
    6. If any test fails due to import issues with @noble/hashes or @noble/curves, the vitest ESM resolution should handle it. If not, check vitest.config.ts `deps.inline` option to force-inline those packages.
    7. Fix any remaining failures until all 94 tests pass.
  </action>
