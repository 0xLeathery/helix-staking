  <action>
    Create `tests/bankrun/phase3.3/securityFixes.test.ts` with vitest syntax (import { describe, it, expect, beforeAll } from "vitest"). Follow the test patterns established in `securityHardening.test.ts` and the Phase 3 utils for bankrun setup.

    **CRIT-1 Tests (zero-bonus deadlock prevention):**
    1. "zero-bonus stake still increments bpd_stakes_distributed counter":
       - Create a stake with minimum amount (0.1 HELIX = 100_000_000 with 9 decimals) and minimum duration (1 day)
       - Run full BPD flow: finalize -> seal -> trigger
       - Verify `bpd_stakes_distributed` equals `bpd_stakes_finalized` after trigger
       - Verify BPD window is cleared (reserved[0] == 0)

    2. "mixed zero-bonus and normal stakes all complete BPD distribution":
       - Create one large stake (100 HELIX, 365 days) and one tiny stake (0.1 HELIX, 1 day)
       - Run BPD flow
       - Verify both stakes are processed (distributed count matches finalized count)
       - Verify large stake received bonus, tiny stake received 0 bonus
       - Verify BPD window clears and unstaking works afterward

    3. "many zero-bonus stakes do not prevent BPD completion":
       - Create 5+ minimum stakes that would all produce bonus == 0
       - Run BPD flow
       - Verify distributed == finalized
       - Verify BPD window is clear
       - Verify all stakers can unstake after BPD completes

    **HIGH-1 Tests (emergency abort_bpd):**
    4. "authority can abort active BPD window":
       - Start BPD finalization (sets window active)
       - Call abort_bpd with authority
       - Verify BPD window is cleared
       - Verify BPD counters are reset to 0
       - Verify BpdAborted event is emitted

    5. "non-authority cannot call abort_bpd":
       - Start BPD finalization
       - Attempt abort_bpd with non-authority keypair
       - Verify it fails with appropriate error

    6. "abort_bpd fails when BPD window is not active":
       - Without starting any BPD process
       - Attempt abort_bpd with authority
       - Verify it fails with BpdWindowNotActive error

    7. "unstaking works after abort_bpd":
       - Create stake, start BPD finalization (window blocks unstaking)
       - Call abort_bpd
       - Attempt to unstake
       - Verify unstake succeeds (window was cleared)

    **HIGH-2 Tests (premature seal prevention):**
    8. "seal_bpd_finalize rejects when no stakes have been finalized":
       - Initialize claim period
       - Set up state as if finalize returned bpd_calculation_complete=true but with bpd_stakes_finalized=0
       - Attempt seal
       - Verify it fails with BpdFinalizationIncomplete error

    9. "seal_bpd_finalize succeeds when stakes have been properly finalized":
       - Create stakes, run finalize_bpd_calculation to completion
       - Call seal_bpd_finalize
       - Verify it succeeds and rate is locked

    **MED-1 Tests (zero-amount finalize BPD window):**
    10. "zero-eligible-stakes finalize clears BPD window":
        - Initialize claim period with no eligible stakes (or stakes that all have 0 share-days)
        - Call finalize_bpd_calculation
        - Verify BPD window is NOT active (reserved[0] == 0)
        - Verify unstaking is not blocked

    Use bankrun's time manipulation (advanceClock) for time-dependent tests. Follow existing patterns from securityHardening.test.ts for setup, keypair generation, and instruction building.
