## CRITICAL Findings (Must Fix)

### CRIT-1: `total_claimed` Can Exceed `total_claimable`, Bricking BPD
**Team:** On-Chain Program
**File:** `free_claim.rs` / `finalize_bpd_calculation.rs`
Speed bonuses (+20%) push `total_claimed` above `total_claimable`. BPD finalize then reverts on `checked_sub`, permanently blocking BPD distribution.
**Fix:** Use `saturating_sub` in finalize, or validate `total_claimable` accounts for max speed bonuses at init.

### CRIT-2: `admin_set_claim_end_slot` — No Bounds Validation
**Team:** On-Chain + Security
**File:** `admin_set_claim_end_slot.rs`
Authority can set `end_slot` to 0 (instant claim termination) or `u64::MAX` (indefinite extension). No event emitted.
**Fix:** Remove for mainnet or add bounds (`new_end_slot > clock.slot`, max extension) + event emission.

### CRIT-3: `admin_set_slots_per_day` — Retroactive Economics Manipulation
**Team:** On-Chain + Security
**File:** `admin_set_slots_per_day.rs`
Changing `slots_per_day` retroactively alters day counting, penalties, loyalty bonuses, inflation, and BPD for ALL existing stakes.
**Fix:** Remove for mainnet or feature-gate. Consider storing per-stake `slots_per_day`.

### CRIT-4: `NEXT_PUBLIC_TEST_WALLET_SECRET` Leaked in Build
**Team:** Frontend
**File:** `providers.tsx`
Test wallet private key embedded in client JS bundle via `NEXT_PUBLIC_` prefix. Any production build with this env var leaks the key.
**Fix:** Remove `NEXT_PUBLIC_` prefix; load via server action or gate behind `NODE_ENV === 'test'`.

### CRIT-5: Transaction Simulation Results Silently Ignored
**Team:** Frontend
**Files:** `useUnstake.ts`, `useClaimRewards.ts`
Both hooks call `simulateTransaction()` but don't check `simulation.value.err`, sending transactions even when simulation indicates failure.
**Fix:** Add `if (simulation.value.err) throw` check (already correct in `useCreateStake`).

### CRIT-6: Indexer Polling Gap >1,000 Signatures = Data Loss
**Team:** Indexer
**File:** `poller.ts`
If indexer is offline during >1,000 transactions, oldest events are silently dropped with no backward pagination.
**Fix:** Implement backward pagination loop in `getSignaturesForAddress` to fetch full gap.

### CRIT-7: No Database Migration Files
**Team:** Indexer
**Path:** `services/indexer/src/db/migrations/`
Schema exists but no versioned migration files. Production deployment risks destructive DDL.
**Fix:** Run `drizzle-kit generate` and commit migration files.

## HIGH Findings (Should Fix)

### HIGH-1: Singleton ClaimConfig — Only One Claim Period Ever
**Team:** On-Chain + Security
No instruction to close ClaimConfig PDA. Protocol limited to one lifetime token distribution event.
**Fix:** Add `close_claim_period` instruction or change PDA seeds to include `claim_period_id`.

### HIGH-2: BPD Window Locks Funds Indefinitely If Authority Unavailable
**Team:** On-Chain + Security
`finalize_bpd_calculation` sets `bpd_window_active = true`, blocking all unstakes. Only authority can clear via `abort_bpd`.
**Fix:** Add time-based permissionless escape hatch (e.g., auto-abort after N days).

### HIGH-3: Loyalty Bonus Creates Unbacked Inflation (+50% Above Declared Rate)
**Team:** On-Chain
Loyalty bonus mints up to 1.5x rewards. Actual inflation can reach ~5.54% vs declared 3.69%.
**Fix:** Document clearly or fund loyalty bonus from existing inflation allocation.

### HIGH-4: No React Error Boundaries in Frontend
**Team:** Frontend
No `error.tsx`, `loading.tsx`, or `not-found.tsx` anywhere. Unhandled errors show raw crash page.
**Fix:** Add error boundaries at root and dashboard levels.

### HIGH-5: Hardcoded `cluster=devnet` in Explorer Links
**Team:** Frontend
Success screen links to `explorer.solana.com?cluster=devnet`. Broken on mainnet.
**Fix:** Derive cluster from `NEXT_PUBLIC_SOLANA_CLUSTER` env var.

### HIGH-6: RPC Proxy Has No Rate Limiting or Method Allowlist
**Team:** Frontend + Indexer
`/api/rpc` forwards any POST to Helius verbatim. Attackers can drain RPC credits.
**Fix:** Add JSON-RPC method allowlist and rate limiting.

### HIGH-7: Indexer Checkpoint Advances Past Failed Events
**Team:** Indexer
If a DB insert fails, the checkpoint still advances, permanently skipping events.
**Fix:** Make event processing + checkpoint update atomic.

### HIGH-8: No Indexer API Rate Limiting
**Team:** Indexer
All 13 endpoints are public with no rate limits. Expensive queries (leaderboard, chart) are DoS-able.
**Fix:** Add `@fastify/rate-limit`.

### HIGH-9: Indexer Anchor Version Mismatch (0.30.x vs 0.31.x)
**Team:** Config & Deploy
Program uses Anchor 0.31.x but indexer uses `^0.30.1`. IDL format changes may cause silent misdecoding.
**Fix:** Bump indexer to `@coral-xyz/anchor@^0.31.0`.

### HIGH-10: No Verifiable Build
**Team:** Config & Deploy
`target/verifiable/` is empty. No evidence of `anchor build --verifiable`.
**Fix:** Produce verifiable build before any deployment.

### HIGH-11: No CI/CD Pipeline
**Team:** Config & Deploy
Zero automated quality gating — no GitHub Actions, no Docker, no build/test on PR.
**Fix:** Set up minimal CI: build, test, anchor verify.
